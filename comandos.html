Comandos git

  • GIT INIT -> Cria um repositório git localmente
  • GIT STATUS -> Apresenta os estados dos arquivos do repositório local. Exemplo: modificado, adicionado, deletado...
  • GIT LOG -> Lista todos os commits efetuados
  • GIT LOG -N 2 -> Lista os dois últimos commits
  • GIT LOG --PARENTS -> Lista os commits associados à seus pais
  • GIT ADD EXEMPLO.HTML -> Adiciona o arquivo exemplo.html na área de stage, o arquivo começa à ser rastreado pelo git
  • GIT ADD . -> Adiciona todos os arquivos na área de stage
  • GIT RM EXEMPLO.HTML -> Remove o arquivo exemplo.html do repositório e adiciona a alteração na área de stage (remoção)
  • GIT MV EXEMPLO.HTML NOVO.HTML-> Renomeia o arquivo exemplo.html para novo.html
  • GIT MV EXEMPLO.JS JS/EXEMPLO.JS -> Move o arquivo exemplo.js para a pasta js OBSERVAÇÃO: para mover o arquivo a pasta já deve estar criada
  • GIT RESET -- INDEX.HTML -> Remove o arquivo index.html da área de stage mantendo sua a alteração no arquivo intacta, o arquivo volta a ser não rastrado.
  • GIT RESET -> Remove todos os arquivos da área de stage , mantendo as alterações intactas nos arquivos.
  • GIT RESET --HARD -> Remove todos os arquivos da área de stage descartando as alterações, voltando ao estado do último commit
  • GIT REVERT --NO-EDIT 666 -> Desfazendo alterações do commit de código 666 OBSERVAÇÃO: No caso de passarmos um código de commit antigo, apenas as alterações feitas naquele commit serão desfeitas.
  • GIT RESET --HARD 666 -> Desfaz as alterações do commit de código 666 OBSERVAÇÃO: OUTRA FORMA DE DESFAZER ALTERAÇÕES COMMITADAS
  • GIT COMMIT -M "COMMIT INICIAL" -> Grava os arquivos da área de stage no repositório local, o parâmetro -m serve para passar uma mensagem
  • GIT COMMIT -A -M "COMMIT INICIAL" -> Adiciona todos os arquivos na área de stage e grava no repositório local, é equivalente ao GIT ADD + GIT COMMIT OBSERVAÇÃO: Não funciona para arquivos não rastreados, ou seja, arquivos que ainda não foram adicionados na área de stage. Exemplo : Arquivos que acabaram de ser criados
  • GIT DIFF EXEMPLO.TXT -> Mostra as mudanças entre o arquivo exemplo.txt alterado(não rastreado) e o commitado anteriormente
  • GIT DIFF --STAGED -> Mostra as mudanças entre um arquivo na área de stage
  • GIT SHOW 666 -> Mostra informações sobre o commit de código 666
  • GIT BRANCH DESIGN -> Cria uma nova branch chamada design
  • GIT BRANCH -V -> Lista as branchs e seus commits associados
  • GIT BRANCH -d DESIGN -> Delete a branch design OBSERVAÇÃO: o parâmetro -d deve estar minúsculo
  • GIT BRANCH -D DESIGN -> Remove branch com commits, os commits são descartados OBSERVAÇÃO: o parâmetro -D deve estar maiúsculo
  • GIT BRANCH --NO-MERGED -> Lista branchs ainda não mescladas à branch atual
  • GIT BRANCH -R -> Lista branchs remotas
  • GIT BRANCH -A -> Lista todas as branchs(remotas e locais)
  • GIT REMOTE ADD ORIGIN -> Cria um repositório git localmente
  • GIT CHECKOUT -- INDEX.HTML -> Desfaz alterações não rastreadas(fora da área de stage) do arquivo index.html
  • GIT CHECKOUT DESIGN -> Troca para a branch design
  • GIT CHECKOUT -B DESIGN -> Cria uma nova branch design e troca a branch
  • GIT CHECKOUT -B DESIGN ORIGIN/MASTER -> Cria a branch local e troca a branch, associando-a a branch remota origin/master
  • GIT PUSH ORIGIN :DESIGN -> Remove a branch design local e remoto.
  • GIT MERGE DESIGN -M "MESCLANDO BRANCHS" -> Mesclando a branch design com a branch atual
  • GIT REBASE DESIGN -> Mesclando a branch desgin com a branch atual (outra forma de mesclar)
  • GIT FETCH ORIGIN -> Obtém commits de uma branch remota que ainda não estão presentes localmente, nesse caso está pegando a branch origin/master
  • GIT PULL -> Sincroniza o repositório com a branch especificada = GIT FETCH + GIT MERGE
  • GIT PULL --REBASE -> Sincroniza o repositório com a branch especificada = GIT FETCH + GIT REBASE
  • GIT PUSH ORIGIN MASTER -> Envia os commits do repositório local para o repositório remoto, neste caso está enviando da branch remota origin/master para a branch master
  • GIT TAG V1.0 -> Cria uma nova tag com o nome V1.0 para o último commit da branch atual
  • GIT TAG V1.0 666 -> Cria uma nova tag com o nome V1.0 para o commit de código 666
  • GIT TAG -A V1.0 -M "NOVA TAG" -> Cria uma tag anotada, ou seja, uma tag com mais informações, comom uma mensagem, criador, data, etc.
  • GIT SHOW -S V1.0 -> Exibe informaões da tag anotada V1.0
  • GIT PUSH ORIGIN V1.0 -> Envia a TAG V1.0 para o repositório remoto
  • GIT PUSH ORIGIN --TAGS -> Envia todas as tags criadas para o repositório remoto
